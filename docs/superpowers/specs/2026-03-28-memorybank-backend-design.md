# MemoryBank Backend Design

## Overview

Integrate MemoryBank (based on [MemoryBank: Enhancing Large Language Models with Long-Term Memory](https://arxiv.org/pdf/2305.10250.pdf)) as a fourth memory retrieval mode (`memorybank`) in thesis-cockpit-memo, alongside existing keyword/llm_only/embeddings modes.

Core capabilities from MemoryBank:

1. **Ebbinghaus Forgetting Curve** - memories decay over time based on `retention = exp(-t / (5 * S))`
2. **Hierarchical Summarization** - per-date summaries and overall summary
3. **Recall-based Memory Strengthening** - retrieved memories get `memory_strength += 1`

## Architecture

### Approach

New file `app/memory/memory_bank.py` implementing `MemoryBankBackend`. Integrated via a new branch in `MemoryModule.search()`.

No FAISS dependency - reuse existing BGE-small-zh-v1.5 + numpy cosine similarity.

## Data Model

### Event Structure (extended)

```json
{
  "id": "20260327120000_a1b2c3d4",
  "content": "提醒明天上午9点会议",
  "type": "reminder",
  "created_at": "2026-03-27T12:00:00",
  "memory_strength": 1,
  "last_recall_date": "2026-03-27",
  "date_group": "2026-03-27"
}
```

- `memory_strength`: initial 1, +1 on each retrieval hit
- `last_recall_date`: updated to current date on retrieval hit
- `date_group`: derived from `created_at`, used for daily grouping

### Summary Storage (`data/memorybank_summaries.json`)

```json
{
  "daily_summaries": {
    "2026-03-27": {
      "content": "用户添加了会议提醒，讨论了日程安排",
      "memory_strength": 1,
      "last_recall_date": "2026-03-27"
    }
  },
  "overall_summary": "用户是一位繁忙的职场人士，主要关注会议和日程管理"
}
```

## Forgetting Curve

```
retention = exp(-t / (5 * S))
```

- `t` = days since last recall (`today - last_recall_date`)
- `S` = `memory_strength`

## Retrieval Flow (memorybank mode)

```
input query
  -> encode query with embedding model
  -> for each event:
       compute cosine similarity
       compute retention = forgetting_curve(days_elapsed, strength)
       final_score = similarity * retention
  -> rank by final_score, return top-k
  -> for each hit event:
       memory_strength += 1
       last_recall_date = today
  -> persist updated events
  -> include daily summaries in results (weight * 0.8)
```

## Summarization Triggers

- **Daily summary**: when events in a date_group >= 5 (configurable via `DAILY_SUMMARY_THRESHOLD`)
- **Overall summary**: when daily_summaries count >= 3 (configurable via `OVERALL_SUMMARY_THRESHOLD`)
- Summaries generated by LLM, stored in `memorybank_summaries.json`
- Summaries participate in retrieval with 0.8x weight multiplier

## MemoryBankBackend Interface

```python
class MemoryBankBackend:
    def __init__(self, data_dir: str, embedding_model: EmbeddingModel, chat_model: ChatModel): ...

    def search(self, query: str, top_k: int = 3) -> list[dict]:
        """Main retrieval with forgetting-weighted scoring"""

    def write_with_memory(self, event: dict) -> str:
        """Write event with memory metadata (strength, last_recall, date_group)"""

    def _forgetting_curve(self, days_elapsed: int, strength: int) -> float:
        """Compute retention probability"""

    def _maybe_summarize(self, date_group: str):
        """Check and trigger daily summarization if threshold met"""

    def _summarize_date(self, date: str) -> str:
        """LLM generates single-day summary"""

    def _summarize_overall(self) -> str:
        """LLM generates overall summary from daily summaries"""
```
