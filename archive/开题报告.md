## 一、本课题研究（设计）的目的：

随着智能汽车与智能座舱的发展，车载系统正从“被动工具型辅助”转向“主动智能型服务”。传统车载备忘录多停留在事件记录与到时提醒层面，缺乏对驾驶环境、用户状态与时空信息的综合理解，难以在复杂驾驶情境下提供低干扰、高价值的主动辅助。
本课题拟面向“车载备忘录从记录工具进化为决策与执行代理”的目标，研究并实现一个具备情境理解、个性化学习与主动决策能力的车载 AI 智能体原型系统，具体目的如下：

1. 研究车载 AI 智能体在时空感知与行为决策中的关键机制，实现个性化、可解释的智能化辅助。
2. 掌握多小模型协同与 Agent 框架设计方法，提升车载 AI 在任务分配、情境理解与用户建模方面的能力。
3. 基于“知行车秘”系统开展实验验证，探索 AI Agent 在驾驶行为提示、日程优化、情绪与生理极限感知等方向的综合应用，推动智能座舱从辅助向伙伴化演进。

## 二、研究（设计）现状和发展趋势：

### 2.1 车载情境感知与信息代理

车载领域的情境感知信息代理核心目标是降低驾驶员交互与认知负荷系统需要在“何时、以何种方式、向谁呈现什么信息”上做出合理决策。
早期工作已探索将用户、车辆与环境上下文纳入统一建模框架，并用概率推理（如贝叶斯网络）推断驾驶员意图，从而实现信息过滤与任务的半/全自动完成。该路线强调可解释性与在不确定信息下的决策能力，为后续将“备忘提醒”升级为“意图理解 + 任务执行”的车载智能体提供了基础方法论。

### 2.2 上下文驱动的多智能体协作与标准化协议

近年来，大模型驱动的 Agent 系统快速发展，复杂任务通常需要多个“专长模块”（日程解析、路线/交通推断、驾驶员状态评估等）协同完成。多智能体协作的关键在于上下文的标准化表达、传递与动态解析，否则容易出现重复推理、信息不一致与决策漂移等问题。
MCP（Model Context Protocol）提出了面向分布式 Agent 架构的“上下文编码/传输/解释”标准化思路，为异构模型之间的轻量通信与互操作提供了参考框架。

### 2.3 驾驶员状态（情境意识）与生理信号融合

车载智能体的主动干预必须以安全为前提。驾驶任务具有强动态性与双任务干扰特征，驾驶员的情境意识（Situation Awareness）与工作负荷直接影响其对提醒/交互的接受度与风险水平。
相关研究表明，利用生理指标对驾驶员状态进行预测具有可行性，可为“何时提醒、提醒强度如何设置、是否需要延后或改用更低干扰通道”提供量化依据。结合可穿戴与车内传感的发展，面向安全的“状态感知 + 自适应干预”将成为车载 Agent 的重要趋势。

### 2.4 情感感知与以人为中心的座舱交互

汽车行业正在从功能堆叠走向以人为中心的交互设计。情感感知与多模态传感（语音语调、面部表情、文本情绪等）使系统能在“同一提醒内容”上做更细致的表达与策略选择（例如在焦虑/疲劳状态下减少打断、更多提供建议与缓冲方案）。
情感感知不应仅作为“识别模块”存在，而应与任务规划、主动提醒策略、可解释反馈形成闭环，成为提升信任与体验的重要手段。

### 2.5 发展趋势总结

综合以上研究现状，车载备忘录智能化的发展趋势可概括为从单点功能（记录/到时提醒）走向“上下文驱动的多智能体协作系统”从被动响应走向主动规划与执行从仅依赖显式输入走向融合时空、交通、驾驶员状态与情绪信号从黑盒生成走向更可解释、可控且隐私友好的落地系统。

## 三、研究（设计）的重点与难点，拟采用的途径（研究手段）：

### 3.1 研究内容与总体方案

本课题拟面向“时空主动提醒 + 个性化学习 + 主动决策与执行”扩展为一个可实验验证的车载 AI Agent 系统。总体方案为

1. 情境建模层构建统一的上下文表示，覆盖时间（当前/未来事件窗口）、空间（当前位置/目的地/POI）、交通（拥堵/事故/ETA）、用户偏好与习惯（从历史备忘与检测结果中抽取）、驾驶员状态（生理/情绪/工作负荷的可选输入）。
2. 任务理解层对自然语言输入与历史记录进行事件抽取、事件聚合与任务归因（会议、出行、采购、联系等），形成可执行的任务列表与约束条件。
3. 策略决策层面向“是否提醒、何时提醒、提醒方式与内容”进行决策。采用“规则/阈值 + 概率推断 + 大模型生成”的混合策略规则用于安全边界与低延迟决策，概率推断用于不确定上下文下的意图估计，大模型用于自然语言解释与多方案生成。
4. 执行与反馈层输出可解释的提醒与建议（如推荐出发时间、路线缓冲、日程冲突提示），并记录用户反馈（接受/忽略/修改），用于个性化策略更新。

### 3.2 重点与难点分析

1. 情境信息不完整与噪声车载场景的时间/地点往往在对话中隐含或模糊，需要在抽取、补全与一致性校验之间取得平衡。
2. 主动提醒的“打断成本”与安全约束提醒过早/过频会造成干扰，提醒过晚又失去价值需要结合驾驶任务优先级与驾驶员状态，建立可控的触发机制。
3. 个性化学习的冷启动与漂移仅靠少量历史数据难以刻画稳定习惯同时习惯会随季节、工作变化而漂移，需要在线更新与遗忘机制。
4. 多模块协同与可解释性事件抽取、习惯识别、时空推断、策略决策之间存在误差传递系统需要提供可追溯的证据链（来自备忘条目、上下文变量、规则命中等）。
5. 隐私保护与数据合规用户日程、联系人、位置与生理信号均属于敏感信息需要在本地化处理、脱敏、最小化存储与可控访问上给出工程方案。

### 3.3 拟采用的研究手段与验证方法

1. 系统工程与原型实现基于现有 Python + Transformers + 本地模型的管线，扩展上下文结构、策略决策与记录反馈模块，形成可运行原型。
2. 混合决策策略将规则（安全/阈值）、概率推断（意图/不确定性）与大模型生成（自然语言解释/多方案建议）结合，提升可控性与鲁棒性。
3. 对照实验与指标评估在驾驶模拟或可控场景下进行对比实验
   a) 无系统
   b) 基础备忘系统（到时提醒/静态提醒）
   c) 完整 Agent 系统（时空+个性化+主动策略）。
   评估任务完成率、提醒及时性、用户体验与认知负荷（可采用量表与访谈）。
4. 数据与案例驱动迭代利用项目已有的事件分割输出与习惯测试集，构造覆盖会议/出行/突发事件/隐私文本的测试用例，迭代提示模板与策略规则。

## 四、研究（设计）进度计划：

拟定进度如下：

1. 第 1–2 周：阅读参考文献，完成需求分析与指标体系定义（提醒及时性、干扰度、任务完成率等）。
2. 第 3–5 周：完成系统总体架构设计与上下文表示方案，梳理模块接口与数据流。
3. 第 6–9 周：在现有备忘录助手与习惯检测基础上，补充时空信息建模与主动提醒策略模块，形成可端到端运行原型。
4. 第 10–12 周：加入突发事件处理与个性化策略更新机制，完善可解释输出与日志记录。
5. 第 13–15 周：设计并完成对照实验（模拟/案例），整理实验数据，完成结果分析与可视化。
6. 第 16–18 周：完成论文撰写与修改，提交终稿与答辩材料。
