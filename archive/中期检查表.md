# 中期检查表

## 题目

知行车秘：具备时空感知与个性化行为学习的车载备忘录 AI Agent

## 英文题目

DrivePal: An In-Vehicle AI Agent with Spatio-Temporal Awareness and Personalized Behavior Learning

## 主要内容和计划进度

主要内容：本课题面向智能座舱场景，设计并实现车载备忘录AI智能体原型，研究驾驶场景下的时空上下文建模与个性化行为学习方法，使系统能够依据驾驶环境与用户偏好主动发起提醒。针对传统车载备忘录仅支持被动记录与到时提醒、缺乏对驾驶情境综合理解的不足，本课题构建覆盖驾驶员状态、时空位置与交通状况的统一上下文表示，通过外部传感数据的请求级注入为情境推理提供可信的事实基础。在记忆建模方面，引入Ebbinghaus遗忘曲线描述记忆强度随时间与召回次数的变化关系，设计分层记忆架构实现对用户长期偏好的积累、衰减与检索，使系统对用户习惯的掌握更贴近人类记忆的运作方式。在策略决策方面，提出确定性安全约束与开放语义推理相结合的混合决策方法，以规则机制保障驾驶场景下的干预安全底线，同时利用大语言模型的上下文理解能力处理规则未覆盖的开放情境。实验评估基于VehicleMemBench基准测试框架，覆盖偏好冲突、条件约束、错误纠正与共指消解四类推理类型，从完全匹配率、字段级与值级指标等维度对比不同记忆策略在车载场景下的检索效果。

计划进度：
1. 前期完成项目背景调研与需求分析，确定四阶段流水线架构与驾驶上下文数据模型；
2. 完成 MemoryBank 记忆系统开发，包括遗忘曲线衰减、三层记忆结构与用户画像模块；
3. 实现规则引擎与策略决策模块，搭建后端服务与 WebUI 调试工作台，形成可端到端运行的原型；
4. 集成 VehicleMemBench 基准测试框架，完成五种记忆策略的适配与对照实验方案设计；
5. 执行全量对照实验，整理实验数据与结论；
6. 完成论文撰写与修改。

## 目前已完成情况

前四阶段计划内容已完成，当前进入全量实验阶段。

1. Agent流水线与安全推理：系统采用四阶段顺序流水线（上下文理解、任务分析、策略决策、执行反馈），规则引擎负责高速公路、疲劳驾驶、工作负荷过载、驻车四类场景的安全约束，支持多规则同时触发时的优先级仲裁与条件合并。规则引擎与大语言模型推理的混合方案兼顾确定性安全边界与开放情境理解。

2. 三层记忆与个性化学习：基于 MemoryBank 模式实现三层结构——底层记录原始交互事件，新交互通过余弦相似度自动归并到已有事件以避免碎片化；中间层对当日事件聚合生成摘要，逐日积累后由 LLM 综合为总体偏好档案；顶层提取用户驾驶风格、习惯与偏好形成画像，多日画像冲突时以最近记录为准。遗忘曲线采用类 Ebbinghaus 指数衰减模型，记忆强度随召回次数递增，使频繁使用的记忆衰减更慢，保留率低于阈值时触发软遗忘（标记但保留数据）。检索阶段三层同步召回，综合余弦相似度与遗忘保留率计算得分，向量搜索无结果时自动回退至关键词搜索。反馈学习按事件类型分别维护提醒权重，用户接受则权重上升、忽略则下降，驱动后续策略决策。

3. 后端服务与调试工具：搭建后端服务与 WebUI 调试工作台，支持驾驶上下文数据（驾驶员状态、时空信息与交通状况）的注入与交互，用于日常功能验证与实验数据采集。

4. 基准测试框架：集成 VehicleMemBench，适配无记忆基线、注入标准答案、键值存储、递归摘要、本项目方案五种策略，共50组数据集500个测试问题，小样本验证已通过。

## 存在的主要问题和解决方案

主要问题：
1. 全量实验尚未执行：全量实验涉及50组数据集共500个测试问题，单次运行耗时较长。
2. 规则引擎覆盖场景有限：当前仅覆盖四类预定义驾驶场景，对城市驾驶、交通拥堵等开放情境的泛化能力不足。
3. 开题规划与实际实现存在偏差：开题阶段规划的情境信息不完整处理、概率推断与多模块误差传递等研究内容，实际开发中改用规则引擎与大语言模型推理的混合方案，技术路线发生调整，论文中需论证这一调整的合理性。

解决方案：
1. 待优化记忆系统实现后，执行全量实验。
2. 在实验中补充一组开放场景测试，量化规则引擎的覆盖边界，论文中结合实验数据讨论这一取舍。
3. 在论文中说明技术路线调整的原因，结合实验数据对比原规划方案与实际方案各自的适用范围。
