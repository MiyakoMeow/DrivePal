# 中期检查表

## 题目

知行车秘：具备时空感知与个性化行为学习的车载备忘录 AI Agent

## 英文题目

DrivePal: An In-Vehicle AI Agent with Spatio-Temporal Awareness and Personalized Behavior Learning

## 主要内容和计划进度

主要内容：本课题设计并实现车载备忘录AI智能体原型，研究驾驶场景下的时空上下文建模与个性化行为学习方法，系统根据驾驶环境与用户偏好主动发起提醒。

情境建模方面，构建覆盖驾驶员状态、时空位置与交通状况的上下文表示，外部传感数据通过请求级注入直接传入流水线，跳过大语言模型推断环节。

记忆建模方面，引入Ebbinghaus遗忘曲线描述记忆强度随时间与召回次数的变化关系，设计分层记忆架构，实现用户长期偏好的积累、衰减与检索。三层结构自底向上分别为原始交互记录、语义聚合的事件摘要与用户画像，检索时综合余弦相似度与遗忘保留率计算得分。

策略决策方面，采用确定性安全约束与大语言模型语义推理相结合的混合方案：规则引擎处理高速公路、疲劳驾驶等可枚举场景的安全约束，语义推理负责规则未覆盖的开放情境。

实验评估基于VehicleMemBench基准测试框架，覆盖偏好冲突、条件约束、错误纠正与共指消解四类推理类型，以完全匹配率、字段级与值级指标对比不同记忆策略在车载场景下的检索效果。

计划进度：
1. 前期完成项目背景调研与需求分析，确定四阶段流水线架构与驾驶上下文数据模型；
2. 完成 MemoryBank 记忆系统开发，包括遗忘曲线衰减、三层记忆结构与用户画像模块；
3. 实现规则引擎与策略决策模块，搭建后端服务与 WebUI 调试工作台，形成可端到端运行的原型；
4. 集成 VehicleMemBench 基准测试框架，完成五种记忆策略的适配与对照实验方案设计；
5. 执行全量对照实验，整理实验数据与结论；
6. 完成论文撰写与修改。

## 目前已完成情况

前四阶段计划内容已完成，当前进入全量实验阶段。

1. Agent流水线与安全推理：系统采用四阶段顺序流水线（上下文理解、任务分析、策略决策、执行反馈），规则引擎负责高速公路、疲劳驾驶、工作负荷过载、驻车四类场景的安全约束，支持多规则同时触发时的优先级仲裁与条件合并。

2. 记忆架构与事件聚合：基于 MemoryBank 模式实现三层结构。底层记录原始交互事件，新交互通过余弦相似度自动归并到已有事件以避免碎片化；中间层对当日事件聚合生成摘要，逐日积累后由 LLM 综合为总体偏好档案；顶层提取用户驾驶风格、习惯与偏好形成画像，多日画像冲突时以最近记录为准。

3. 遗忘曲线与检索：遗忘曲线采用类 Ebbinghaus 指数衰减模型，记忆强度随召回次数递增，频繁使用的记忆衰减更慢，保留率低于阈值时触发软遗忘（标记但保留数据）。检索阶段三层同步召回，按相似度与遗忘保留率的加权得分排序，向量搜索无结果时自动回退至关键词搜索。

4. 反馈学习：按事件类型分别维护提醒权重，用户接受则权重上升、忽略则下降，驱动后续策略决策。

5. 后端服务与调试工具：搭建后端服务与 WebUI 调试工作台，支持驾驶上下文数据的注入与交互，用于日常功能验证与实验数据采集。

6. 基准测试框架：集成 VehicleMemBench，适配无记忆基线、注入标准答案、键值存储、递归摘要、本项目方案五种策略，共50组数据集500个测试问题，小样本验证已通过。

## 存在的主要问题和解决方案

主要问题：
1. 全量实验尚未执行：全量实验涉及50组数据集共500个测试问题，单次运行耗时较长。小样本（2组数据集20题）验证已通过，待记忆系统参数优化完成后启动全量实验，预计第14周完成。
2. 规则引擎覆盖场景有限：当前仅覆盖高速公路、疲劳驾驶、工作负荷过载、驻车四类场景的安全约束规则，城市驾驶与交通拥堵两类已定义场景尚未配置专属规则。
3. 开题规划与实际实现存在偏差：开题阶段规划的情境信息不完整处理、概率推断与多模块误差传递等研究内容，实际开发中改用规则引擎与大语言模型推理的混合方案，技术路线发生调整。

解决方案：
1. 优化记忆系统参数后启动全量实验，预计第14周完成。
2. 在实验中补充一组开放场景测试，量化规则引擎的覆盖边界，论文中结合实验数据讨论这一取舍。
3. 在论文中说明技术路线调整的原因，结合实验数据对比原规划方案与实际方案各自的适用范围。

下一步计划：
1. 优化记忆系统参数，补充城市驾驶与交通拥堵场景的规则配置；
2. 执行五种记忆策略的全量对照实验，整理实验数据与初步结论；
3. 完成论文各章节撰写与修改；
4. 论文定稿，准备答辩材料。
