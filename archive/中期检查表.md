# 中期检查表

## 题目

知行车秘：具备时空感知与个性化行为学习的车载备忘录 AI Agent

## 英文题目

DrivePal: An In-Vehicle AI Agent with Spatio-Temporal Awareness and Personalized Behavior Learning

## 主要内容和计划进度

本课题设计并实现一个车载备忘录AI智能体原型，研究驾驶场景下的情境推理与用户偏好学习方法，使系统可依据时空上下文主动发起提醒。计划进度如下：

1. 前期完成项目背景调研与需求分析，确定四阶段流水线架构与驾驶上下文数据模型；
2. 完成 MemoryBank 记忆系统开发，包括遗忘曲线衰减、三层记忆结构与用户画像模块；
3. 实现规则引擎与策略决策模块，搭建后端服务与 WebUI 调试工作台，形成可端到端运行的原型；
4. 集成 VehicleMemBench 基准测试框架，完成五种记忆策略的适配与对照实验方案设计；
5. 执行全量对照实验，整理实验数据与结论；
6. 完成论文撰写与修改。

## 目前已完成情况

前四阶段计划内容已完成，当前处于全量实验准备阶段。

1. Agent流水线：系统采用四阶段顺序流水线（上下文理解、任务分析、策略决策、执行反馈），规则引擎覆盖高速公路、疲劳驾驶、工作负荷过载、驻车四类场景的安全约束，支持多规则同时触发时的优先级仲裁与条件合并；

2. 记忆系统：基于 MemoryBank 模式实现三层结构——底层记录原始交互，中间层完成事件聚合摘要，顶层提取用户画像。遗忘曲线控制记忆衰减，反馈学习根据用户对提醒的接受或忽略行为动态调整提醒权重；

3. 后端服务：通过 GraphQL API 提供数据交互接口，查询接口包括历史记录、实验报告、场景预设，变更接口包括备忘录处理、反馈提交、场景预设增删。驾驶上下文数据模型覆盖驾驶员状态、时空信息与交通状况，外部数据通过接口注入；

4. 基准测试：VehicleMemBench 已完成集成，适配了无记忆基线、注入标准答案、键值存储、递归摘要、本项目方案五种策略，共50组数据集500个测试问题，小样本验证已通过；

5. WebUI 工作台：支持场景配置、工作流四阶段调试输出与交互反馈测试，用于功能验证与演示。

## 存在的主要问题和解决方案

1. 全量实验尚未执行
   基准测试框架已集成且小样本验证通过，全量实验涉及50组数据集共500个测试问题，单次运行耗时较长。后续将拆分为多个子任务并行执行，并优化模型调用方式压缩运行时间，完成后整理实验结果。

2. 开题规划与实际实现存在偏差
   开题阶段规划的情境信息不完整处理、多模块误差传递等内容，实际开发中改用规则引擎与大语言模型推理的混合方案。功能上覆盖了四类驾驶场景的安全需求，但仅支持预定义场景类型，泛化能力有限。后续在论文中如实讨论这一取舍。

3. 驾驶员状态感知仅预留接口
   情绪与疲劳度字段已定义，但系统内部未实现情绪识别或疲劳检测的计算逻辑，运行时依赖外部数据注入。实验阶段通过场景预设模拟不同驾驶员状态，论文中将该部分列为待延伸工作。
